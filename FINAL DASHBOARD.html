Step 01
Data Engineering
Ingest CSV & NDJSON via COPY INTO. Implement RAW → VALIDATED → CURATED layers. Deduplicate by natural keys, normalise timestamps to UTC, standardise geo precision and enforce range constraints.
COPY INTO Dedup UTC Normalize SCD Type-2
Step 02
Data Quality & Validation
Referential integrity checks across rentals ↔ bikes ↔ stations ↔ users. Enforce temporal validity (end_time ≥ start_time), battery range 0–100, distance_km ≥ 0. Write DQ exceptions to audit log.
DQ Rules Ref Integrity Audit Log MERGE Upserts
Step 03
Anomaly Detection
PL/SQL stored procedures flag: repeated ultra-short trips, GPS mismatch (telemetry vs station), unrealistic speed/distance, e-bike zero battery starts, device reuse across accounts, rides outside geofence.
Stored Procs Risk Rules GPS Mismatch Geofence
Step 04
Governance & Automation
Snowflake Streams + Tasks for incremental processing. Dynamic masking for PII (email, phone, GPS). Row-level security by region/city_zone. Audit logging per batch — row counts, DQ failures, task runtimes.
Streams + Tasks Dynamic Masking RLS Lineage
